# Medical Workflow Lessons

This local file records problems, causes, fixes, and proposed workflow
improvements found while running the medical map update workflow.

Rules:

- Codex may add problem/fix notes after each run when they only document what
  happened and how it was resolved.
- Workflow improvements must be written under "Proposed Improvements" first.
- Codex must ask the user to review and approve proposed rule changes before
  changing `medical-data-workflow.md`, `RUNBOOK.md`, or other workflow rules.
- Do not store secrets, tokens, credentials, real public CSV URLs, full raw CSV
  dumps, private user data, or full feedback snapshots here.

## Problem/Fix Log

Use this format:

```md
### YYYY-MM-DD - Short problem title

- Context:
- Symptom:
- Cause:
- Fix:
- Prevention:
- Affected files or commands:
```

