# Notes Style Guide

Write `notes` for public map users, not for internal maintainers.

## Goals

- Concise.
- Accurate.
- Actionable.
- Cautious when information can become stale.
- Free of internal workflow traces.

## Do Not Include

Do not put these phrases or ideas into public notes:

- 截图显示
- 文字被截断
- 反馈者表示
- 来源用户
- 群友说
- 医生姓名来自截图文字
- OCR
- AI 判断
- 建议后续核对医院挂号页

Do not directly copy raw feedback. Rewrite it into neutral, public-facing
language.

## Preferred Wording

Acceptable examples:

- 建议就诊前核对医院挂号信息。
- 确诊检查需另行预约。
- 成人就诊前建议确认是否接诊。
- 可作为开药分流选择，建议携带既往确诊材料、处方记录和检查资料。
- 有反馈提到院内可咨询相关开药情况，建议就诊前或现场确认库存与处方要求。

When describing medication inventory, accepted age ranges, departments, or
registration channels, avoid permanent claims. Prefer `可咨询`, `建议确认`, and
`有反馈提到`.

## Rewrite Pattern

1. Identify what a map user can do with the information.
2. Remove source mechanics, screenshot/OCR language, and user identity.
3. Keep concrete appointment, department, material, or capacity information.
4. Add a confirmation sentence when the information can change.
